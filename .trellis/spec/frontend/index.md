# 前端开发规范

> ai-home 前端开发最佳实践。记录真实项目约定，而不是泛用 React 教程。

---

## 概述

ai-home 是一个专业、克制、表格优先的比价工具。前端实现要服务于快速扫描、比较来源、查看库存与更新时间，而不是营销展示。

前端 UI 工艺必须同时遵守 Impeccable Skill 的产品型界面规范。Trellis 负责项目工程边界、数据契约和实现清单；Impeccable 负责界面信息架构、视觉克制、可访问性、响应式和组件质感。两者冲突时，先守住 Trellis 的数据与工程安全边界，再在该边界内按 Impeccable 打磨体验。

## 快速红线

- 不做营销式首页、装饰性英雄区、渐变大背景或卡片堆叠。
- 桌面端优先表格和紧凑筛选；移动端用紧凑列表。
- 库存状态只显示 `有货` / `缺货`，不要发明可信度、多层状态或评分。
- 缺货报价可以出现在详情里，但不能作为外层最低价。
- 客户端请求必须有 race guard、错误保留已有数据、分页不能污染当前筛选。
- 中文按钮、标签、表格列必须在 390px 宽度下检查不溢出。
